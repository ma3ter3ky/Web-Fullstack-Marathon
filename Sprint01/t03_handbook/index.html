PROTOCOLS

HTTP methods

  • GET - use this type of request to retrieve resource representation/information only – and not to modify it in any way

  • POST - used to create a new resource into the collection of resources

  • PUT - primarily used to update an existing resource

  • DELETE - used to delete resources (identified by the Request-URI)

  • PATCH - HTTP PATCH requests are to make partial updates on a resource

HTTP status codes

  • 1xx: Informational – Example: 100 Continue

  • 2xx: Success – Example: 201 Created

  • 3xx: Redirection – Example: 301 Moved Permanently

  • 4xx: Client Error – Example: 404 Not Found

  • 5xx: Server Error – Example: 500 Internal Server Error

DNS list by types

  • A - (address record) - specifies an IPv4 address for a given host.

  • AAAA - (IPv6 address record) - specifies an IPv6 address for a given host.

  • CNAME - (canonical name record) - specifies a domain alias.

  • HINFO - (Host information record) - provides system-specific details.

  • ISDN - (Integrated Services Digital Network record) - specifies ISDN addresses.

  • MX - (Mail exchange record) - specifies mail exchange servers.

  • NS - (Name server record) - specifies an authoritative name server.

  • PTR - (Pointer record) - used to reverse DNS resolution.

  • SOA - (Start of Authority record) - specifies DNS zone details.

  • TXT - (Text record) - used to store arbitrary text data.

Identifying Google's IP

  1. Open Terminal

  2. Paste and run the command: nslookup google.com

  3. Result example:
    • Name: google.com.ua
    • Address: 216.58.215.99


  4. Repeat steps 1 and 2 to identify another IP
